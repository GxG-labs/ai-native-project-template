# Content Pillars

## Scope

Main thematic categories that organize content strategy.  
Every new content should fit into exactly one pillar.  
If content doesn't fit any pillar → discuss adding a new one (don't stretch existing definitions).

## Structure

- **Name** (required): 2–4 words, distinct from others
- **Purpose** (required): One sentence — what outcome does this drive?
- **What fits** (required): 2–3 examples of content that belongs here

## Rules for Adding

Before adding a pillar:
1. Do we have 3+ pieces of content that fit this and nothing else?
2. Does it avoid overlapping with existing pillars?
3. Does it support the project's main job (see `PROJECT.md`)?

If yes to all three, add it using the structure above.

## Examples

### Valid: Thought Leadership

- **Purpose:** Establish credibility through original research and opinions
- **What fits:** Original analysis, contrarian takes on trends, behind-the-scenes decisions

### Valid: Practical Guides

- **Purpose:** Help users solve specific problems with step-by-step instructions
- **What fits:** Tutorials, checklists, troubleshooting guides

### Invalid (too vague, would overlap)

- **Purpose:** Interesting content about our industry
- **What fits:** Anything we think is cool
