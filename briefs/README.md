# Briefs

Use this folder for project intent, task requests, requirements, client requests, research questions, and other starting points.

A brief is input. It describes what was asked for, not necessarily what the final system became.

Think of `briefs/` as the meaning input:

```text
briefs/ = what needs to be done and why
```

Do not use this folder as a drop zone for source files. Put task files in `workbench/input/` unless they already belong in a more specific durable location.
